# Chapter 12: Glossary

| Term | Definition |
|------|------------|
| Kata | A small, well-defined programming exercise for deliberate practice |
| Rover | The simulated vehicle navigating the grid |
| Command | A single instruction: `F` (forward), `B` (backward), `L` (turn left), `R` (turn right) |
| Heading | The direction the rover faces: `N`, `S`, `E`, `W` |
| Grid | The bounded 2D surface the rover moves on |
| Obstacle | A blocked cell the rover cannot enter |
| RoverState | Immutable snapshot of the rover's current position `(x, y)` and heading |
| Bounded Context | A logical boundary within which a domain model is consistent |
| ADR | Architecture Decision Record — documents a significant design choice |
| arc42 | A template for documenting software architecture in 12 chapters |
| C4 Model | A hierarchical diagramming approach: Context, Container, Component, Code |
| PlantUML | A text-based tool for generating UML and C4 diagrams |
