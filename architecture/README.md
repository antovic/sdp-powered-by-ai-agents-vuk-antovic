# arc42 Chapter Summaries

- **Ch 1 - Introduction and Goals**: Defines what the system does and what quality goals it must meet.
- **Ch 2 - Constraints**: Lists technical and organisational constraints that limit design choices.
- **Ch 3 - Context and Scope**: Shows the system boundary and its interactions with external actors.
- **Ch 4 - Solution Strategy**: Summarises the key decisions made to meet the goals and constraints.
- **Ch 5 - Building Block View**: Describes the static structure of the system and its components.
- **Ch 6 - Runtime View**: Shows how components interact at runtime for key scenarios.
- **Ch 7 - Deployment View**: Describes how the system is deployed onto infrastructure.
- **Ch 8 - Cross-cutting Concepts**: Covers concerns that apply across the whole system like logging and error handling.
- **Ch 9 - Architecture Decisions**: Records important decisions as ADRs with context and rationale.
- **Ch 10 - Quality Requirements**: Defines measurable quality scenarios the system must satisfy.
- **Ch 11 - Risks and Technical Debts**: Lists known risks and shortcuts taken during development.
- **Ch 12 - Glossary**: Defines key terms used throughout the architecture documentation.
